# Goaldone Projekt

Das ist die finale Implementierung des Goaldone Projekts. Wir nutzen ein Monorepo für unser Projekt, 
da so keine OpenAPI.yaml hin und herkopiert werden muss. Die OpenAPI.yaml ist unsere Single Source of Truth
und wird benutzt um sowohl im Frontend als auch im Backend automatisch Interfaces und DTOs zu generieren.
