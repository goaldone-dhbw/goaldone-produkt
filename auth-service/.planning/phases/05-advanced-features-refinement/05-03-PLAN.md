# Phase 5.3: Email Templates & German Language Support Refinement

**Status:** Ready for execution  
**Priority:** Medium-High (UX/polish)  
**Estimated Effort:** 25-30 hours

---

## Goal

Refine all email templates and user-facing pages with visual polish, German language translation, and accessibility improvements (plain-text versions). Create new account linking confirmation email template.

---

## Scope

**In Scope:**
- German translation of all user-facing email templates
- German translation of all user-facing HTML pages
- Visual polish: HTML layout refinement, colors, typography per design system
- Plain-text versions of all HTML emails (accessibility, fallback rendering)
- New account linking confirmation email template (integrated with Phase 5.1)
- Email client testing (Gmail, Outlook, Apple Mail, etc.)
- Consistent German terminology (glossary document)

**Out of Scope:**
- Multi-language i18n infrastructure (English, French, etc. — deferred to future phase)
- Email service provider (Mailgun, SendGrid) configuration
- Automated email testing frameworks (manual testing only)
- Email list unsubscribe management (not applicable for transactional emails)

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Email client rendering inconsistencies | Poor user experience | Test early and often; provide plain-text fallback; document known issues |
| German translation quality | User confusion | Use professional translator or native speaker review; document terminology for consistency |
| Variable injection errors | Broken emails, support burden | Comprehensive unit tests; test with sample data before deployment |
| Multipart email formatting broken | Users see both HTML and plain-text | Test before deployment; verify Content-Type headers set correctly |
| GoaldoneTheme CSS vars not available at template render time | Styling broken | Verify CSS vars in base template; use fallback colors if needed |
| Email size too large | Slow delivery, rejection by some providers | Minimize images, use data URIs for small images, compress CSS |
