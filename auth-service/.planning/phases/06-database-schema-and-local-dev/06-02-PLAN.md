# Phase 6.2: Enable Local Development Without Redis

**Status:** Ready for planning  
**Priority:** High (enables local dev testing)  
**Estimated Effort:** 12-16 hours

---

## Goal

Enable local development and testing of the auth-service without requiring an external Redis instance. Provide fallback session storage mechanism that works out-of-the-box for developers.

---

## Problem Statement

**Current State:** Application requires Redis for Spring Session JDBC session storage, making local development setup complex and dependent on external services.

**Developer Pain Points:**
- Redis setup overhead for local testing
- Difficult to test on machines without Redis installed
- CI/CD pipeline complexity for test environments
- Session loss if Redis is not available

**Solution Approach:** 
- Configure Spring Session to support multiple backends
- Use in-memory H2 database as fallback for local development
- Enable conditional configuration based on environment
- Keep Redis as production option

---

## Scope

**In Scope:**
- Add H2 database dependency for local session storage
- Configure Spring Session JDBC with H2 for dev profile
- Create alternate session configuration for local development
- Update application.yaml with environment-specific profiles
- Document local development setup without Redis
- Provide docker-compose for easy Redis startup (optional)
- Test local flows with H2 session storage

**Out of Scope:**
- Redis cluster configuration (production concern)
- Session migration between storage backends
- Performance comparison (H2 vs Redis)
- Custom session serialization

**Production Configuration (application.yaml):**
```yaml

    store-type: jdbc

      table-name: SPRING_SESSION

    host: ${REDIS_HOST:localhost}
    port: ${REDIS_PORT:6379}
```

**Dev Configuration (application-dev.yaml):**
```yaml
spring:
  datasource:
    # H2 in-memory or file-based
    url: jdbc:h2:mem:auth_session;DB_CLOSE_DELAY=-1
    # or: jdbc:h2:file:./target/auth_session
    driver-class-name: org.h2.Driver
    username: sa
    password: 
  
  session:
    store-type: jdbc
    jdbc:
      table-name: SPRING_SESSION
      
  h2:
    console:
      enabled: true  # Optional: enable H2 console for dev
```

**Success Criteria:**
- Configuration profiles clearly separated
- Environment variables for secrets
- Dev profile uses H2, prod/default uses Redis
- All configuration options documented
