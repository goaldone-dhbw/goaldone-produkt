# Plan 06.1-02 Summary: Docker-Compose — Auth-Service & Postgres-Auth Services

## Status: COMPLETE

## What Was Built

Updated `docker/dev/docker-compose.yaml` and `docker/prod/docker-compose.yaml` to deploy auth-service as a standalone containerized service with its own PostgreSQL instance.

### Dev docker-compose.yaml changes:
1. **postgres-auth-dev** service — PostgreSQL for auth-service, port 5432, `postgres-auth-dev-data` volume, health check on `pg_isready`
2. **auth-service-dev** service — ghcr.io auth-service image, depends on `postgres-auth-dev (service_healthy)`, health check on port 9000 actuator endpoint, injects `AUTH_SERVICE_RSA_PRIVATE_KEY` and `AUTH_SERVICE_JWT_ISSUER_URI` from .env
3. **backend-dev** `depends_on` extended to include `auth-service-dev: condition: service_healthy` (startup order per D-03)
4. Removed all `ZITADEL_*` env vars from backend-dev; replaced with `AUTH_SERVICE_JWT_ISSUER_URI`
5. Updated frontend-blue/green: replaced `ZITADEL_CLIENT_ID`/`ZITADEL_ISSUER_URI` with `OIDC_CLIENT_ID`/`OIDC_ISSUER_URI` pointing to auth-service
6. Added `postgres-auth-dev-data:` to volumes section

### Prod docker-compose.yaml changes:
Mirror of dev with `-prod` suffixes and `prod` Spring profile.

### .env.example updates:
- **docker/dev/.env.example** — Removed all `ZITADEL_*` variables; added full `AUTH_SERVICE_*` section with dev issuer URI (`http://auth-service-dev:9000`), plus `FRONTEND_CLIENT_ID`
- **docker/prod/.env.example** — Same, with prod issuer URI (`https://auth.goaldone.de`)
