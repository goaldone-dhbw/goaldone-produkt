  <name>Task 1: Update docker-compose.yaml (dev) — Add postgres-auth and auth-service Services</name>
  <files>docker/dev/docker-compose.yaml</files>
  <action>
    Add two new services to the dev docker-compose file: postgres-auth-dev (PostgreSQL for auth-service) and auth-service-dev (auth-service container).
    
    **Placement:** After `postgres-dev` service and before `backend-dev` service (maintains logical ordering).
    
    **postgres-auth-dev service:**
    ```yaml
    postgres-auth-dev:
      image: postgres:17-alpine
      restart: unless-stopped
      environment:
        POSTGRES_DB: ${AUTH_SERVICE_DB_NAME:-auth_service}
        POSTGRES_USER: ${AUTH_SERVICE_DB_USER:-auth_service}
        POSTGRES_PASSWORD: ${AUTH_SERVICE_DB_PASSWORD}
      volumes:
        - postgres-auth-dev-data:/var/lib/postgresql/data
      networks:
        - app-dev
      healthcheck:
        test: ["CMD-SHELL", "pg_isready -U ${AUTH_SERVICE_DB_USER:-auth_service}"]
        interval: 10s
        timeout: 5s
        retries: 5
    ```
    
    Notes:
    - Image: postgres:17-alpine (same as backend Postgres)
    - Volume name: postgres-auth-dev-data (distinct from postgres-dev-data per D-02)
    - Health check: Same pattern as postgres-dev
    - Environment: Uses AUTH_SERVICE_DB_* variables from .env
    
    **auth-service-dev service:**
    ```yaml
    auth-service-dev:
      image: ghcr.io/goaldone-dhbw/goaldone-auth-service:${AUTH_SERVICE_VERSION}
      restart: unless-stopped
      environment:
        SPRING_PROFILES_ACTIVE: dev
        SPRING_DATASOURCE_URL: jdbc:postgresql://postgres-auth-dev:5432/${AUTH_SERVICE_DB_NAME:-auth_service}
        SPRING_DATASOURCE_USERNAME: ${AUTH_SERVICE_DB_USER:-auth_service}
        SPRING_DATASOURCE_PASSWORD: ${AUTH_SERVICE_DB_PASSWORD}
        AUTH_SERVICE_RSA_PRIVATE_KEY: ${AUTH_SERVICE_RSA_PRIVATE_KEY}
        AUTH_SERVICE_JWT_ISSUER_URI: ${AUTH_SERVICE_JWT_ISSUER_URI}
      networks:
        - app-dev
      depends_on:
        postgres-auth-dev:
          condition: service_healthy
      healthcheck:
        test: ["CMD-SHELL", "curl -sf http://localhost:8080/oauth2/jwks > /dev/null || exit 1"]
        interval: 15s
        timeout: 10s
        retries: 5
        start_period: 30s
    ```
    
    Notes:
    - Image: ghcr.io/goaldone-dhbw/goaldone-auth-service:VERSION (GHCR reference, auto-pulled by Docker)
    - Spring profile: dev (matches backend pattern)
    - Database URL: jdbc:postgresql://postgres-auth-dev:5432/... (hostname must match postgres-auth-dev service name for Docker DNS resolution)
    - Environment variables: AUTH_SERVICE_RSA_PRIVATE_KEY and JWT_ISSUER_URI injected from .env (per D-05, D-09, D-10)
    - depends_on: postgres-auth-dev with condition: service_healthy (ensures Postgres is ready — per D-03)
    - Network: app-dev (internal, not exposed via Traefik)
    - Health check: JWKS endpoint (per D-07) — must succeed before backend starts
    
    **Validation:** YAML syntax valid, service names match between depends_on and networks, volume names are unique.
  </action>
