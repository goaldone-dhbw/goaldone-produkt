# Plan 06.1-03 Summary: deploy.sh — Auth-Service Startup Orchestration

## Status: COMPLETE

## What Was Built

Extended `scripts/deploy.sh` to orchestrate auth-service startup with health checks and JWKS validation before backend initialization.

### Changes to scripts/deploy.sh:

1. **Updated header** — Script now accepts 4 parameters: `<stage> <backend_version> <frontend_version> <auth-service_version>`
2. **AUTH_SVC_VER=$4** — Fourth positional argument extracted and printed at startup
3. **Image pull** — Added `docker pull ghcr.io/goaldone-dhbw/goaldone-auth-service:$AUTH_SVC_VER`
4. **Postgres-Auth section** — Starts `postgres-auth-$STAGE`, polls health status (12 attempts × 5s = 60s max), exits on failure
5. **Auth-Service section** — Updates `AUTH_SERVICE_VERSION` in .env via sed, starts `auth-service-$STAGE`, polls health (24 attempts × 5s = 120s max), validates JWKS at `http://auth-service-${STAGE}:9000/oauth2/jwks`
6. **Backend section** — Added comment documenting auth-service dependency (docker-compose depends_on enforces it)
7. **Deployment summary** — Added `Auth-Service version: $AUTH_SVC_VER` between frontend and active slot lines

### Startup Order:
```
postgres-auth → auth-service (health check + JWKS validated) → postgres → backend → frontend (blue-green)
```
