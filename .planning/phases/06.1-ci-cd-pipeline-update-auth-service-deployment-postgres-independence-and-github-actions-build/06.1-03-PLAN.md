  <files>scripts/deploy.sh</files>

    Insert a new section after the initial docker pull commands and before the existing "Postgres: Ensure database is running" section. This new section ensures postgres-auth is healthy (separate from postgres-backend per D-02).
    
    **Placement:** After docker pull commands (lines 18-23), add a new section before the existing postgres-dev section.
    
    **New section (approximately 20 lines):**
    ```bash
    # ============================================================================
    # Postgres-Auth: Ensure auth-service database is running
    # ============================================================================
    echo "Ensuring postgres-auth is running..."
    docker compose -f "$DIR/docker-compose.yaml" up -d --no-deps "postgres-auth-$STAGE"

    echo "Waiting for postgres-auth health check..."
    max_attempts=12
    attempt=0
    while [ $attempt -lt $max_attempts ]; do
      STATUS=$(docker inspect --format='{{.State.Health.Status}}' \
        "$(docker compose -f "$DIR/docker-compose.yaml" ps -q "postgres-auth-$STAGE")" 2>/dev/null || echo "starting")
      echo "  Attempt $((attempt + 1))/$max_attempts - Status: $STATUS"
      [ "$STATUS" = "healthy" ] && break
      attempt=$((attempt + 1))
      sleep 5
    done

    if [ $attempt -eq $max_attempts ]; then
      echo "❌ Postgres-Auth health check failed after $max_attempts attempts"
      exit 1
    fi
    echo "✓ Postgres-Auth is healthy"
    ```
    
    **Why:** Auth-service needs its own Postgres instance (D-02). Must be healthy before auth-service starts. Uses same health check pattern as postgres-dev (lines 28-46 of current script).
    
    **Validation:** postgres-auth-$STAGE is started and health check passes or script exits with error.

  <name>Task 3: Add Auth-Service Startup and JWKS Validation Stage</name>
  <files>scripts/deploy.sh</files>
  <action>
    Insert a new section after postgres-auth health check (and before or after postgres-dev, but before backend section). This section pulls the auth-service image, updates the .env version, starts the container, waits for health check, and validates the JWKS endpoint.
    
    **Placement:** After postgres-auth section, after postgres-dev section, before backend section.
    
    **New section (approximately 35 lines):**
    ```bash
    # ============================================================================
    # Auth-Service: Pull image, start, and validate JWKS endpoint
    # ============================================================================
    echo "Pulling auth-service image: ghcr.io/goaldone-dhbw/goaldone-auth-service:$AUTH_SVC_VER"
    docker pull ghcr.io/goaldone-dhbw/goaldone-auth-service:$AUTH_SVC_VER

    echo "Updating auth-service to version $AUTH_SVC_VER..."
    sed -i "s/^AUTH_SERVICE_VERSION=.*/AUTH_SERVICE_VERSION=$AUTH_SVC_VER/" "$ENV_FILE"
    docker compose -f "$DIR/docker-compose.yaml" up -d --no-deps "auth-service-$STAGE"

    echo "Waiting for auth-service health check..."
    max_attempts=24
    attempt=0
    while [ $attempt -lt $max_attempts ]; do
      STATUS=$(docker inspect --format='{{.State.Health.Status}}' \
        "$(docker compose -f "$DIR/docker-compose.yaml" ps -q "auth-service-$STAGE")" 2>/dev/null || echo "starting")
      echo "  Attempt $((attempt + 1))/$max_attempts - Status: $STATUS"
      [ "$STATUS" = "healthy" ] && break
      attempt=$((attempt + 1))
      sleep 5
    done

    if [ $attempt -eq $max_attempts ]; then
      echo "❌ Auth-Service health check failed after $max_attempts attempts"
      exit 1
    fi
    echo "✓ Auth-Service is healthy"

    # Validate JWKS endpoint is reachable (D-07: JWKS Endpoint Health Check)
    echo "Validating JWKS endpoint..."
    if curl -sf "http://auth-service-${STAGE}:8080/oauth2/jwks" > /dev/null; then
      echo "✓ JWKS endpoint is reachable"
    else
      echo "❌ JWKS endpoint validation failed"
      exit 1
    fi
    ```
    
    **Why:** Auth-service must be running and JWKS endpoint must be reachable before backend starts (D-06, D-07). Docker health check validates container status, curl validates endpoint accessibility.
    
    **Key Points:**
    - Version update: sed on AUTH_SERVICE_VERSION in .env (matches pattern for backend/frontend)
    - Health check timeout: 24 attempts * 5s = 120s (matches backend pattern, allows for startup time)
    - JWKS validation: curl to /oauth2/jwks endpoint (per D-07 specifics)
    - Exits with error if any stage fails (prevents proceeding with broken auth-service)
    
    **Validation:** Auth-service container healthy and JWKS endpoint reachable.
  </action>
