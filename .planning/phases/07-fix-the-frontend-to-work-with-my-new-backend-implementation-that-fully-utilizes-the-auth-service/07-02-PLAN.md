# Phase 07-02 Plan: Auth Service Token Integration & OIDC Verification

## Goal
Verify that the complete OIDC login flow works end-to-end with the live auth-service backend, confirm token format matches expectations, validate JWT claim extraction (authorities, user_id, orgs), and ensure multi-org provisioning initializes correctly on first login.

## Dependencies
- Plan 07-01 complete (API client regenerated, breaking changes documented)
- Auth-service running and accessible locally
- Backend running with auth-service integration complete

## Success Criteria
1. Complete OIDC flow verified end-to-end:
   - env.js configuration loads correctly
   - OIDC discovery document resolves from auth-service
   - Authorization code flow with PKCE succeeds
   - Token exchange with auth-service succeeds
   - JWT token is decoded and claims are extracted
2. Token format validation:
   - JWT contains `authorities` claim as array of role strings (e.g., ["ROLE_USER", "ROLE_ADMIN"])
   - JWT contains `user_id` claim (UUID string)
   - JWT contains `orgs` claim as array of objects: `[{id, slug, role}, ...]`
3. AuthService token processing works correctly:
   - `AuthService.initialize()` completes successfully
   - `AuthService.getUserRoles()` returns extracted authorities
   - `AuthService.getOrganizations()` returns orgs array with first org as default
4. Multi-org provisioning:
   - OrgContextService initializes with all orgs from JWT
   - First org is automatically selected as default
   - Dialog and settings org context are cleared on init
5. Token refresh works on-demand (5-minute buffer)
6. authInterceptor injects Bearer token and X-Org-ID header correctly

### Task 3: Decode and Validate JWT Token
**In browser console or test file:**

1. In AuthService or test, decode the token and inspect claims:
   ```typescript
   // In auth.service.ts or test
   const token = this.oAuthService.getAccessToken();
   const decoded = this.decodeJwtToken(token);
   console.log('Decoded token:', decoded);
   console.log('authorities:', decoded.authorities);
   console.log('user_id:', decoded.user_id);
   console.log('orgs:', decoded.orgs);
   ```
2. Verify the token contains:
   - `authorities`: Array of strings like `["ROLE_USER"]` or `["ROLE_ADMIN", "ROLE_USER"]`
   - `user_id`: UUID string (36 characters with hyphens)
   - `orgs`: Array with at least one object: `[{id: "uuid", slug: "org-name", role: "ROLE_ADMIN" | "ROLE_USER"}, ...]`
3. If token format differs, document the actual format and update claims in AuthService if needed
