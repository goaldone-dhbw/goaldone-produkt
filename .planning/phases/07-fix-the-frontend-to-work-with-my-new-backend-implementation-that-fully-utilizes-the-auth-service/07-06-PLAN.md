# Phase 07-06 Plan: Manual E2E Testing & Final Validation

## Goal
Manually verify that all core user workflows function end-to-end with the auth-service-based backend. Validate login flow, multi-org context management, member management operations, and error handling. Ensure frontend integrates seamlessly with backend and all user-facing features work as expected. Document results and prepare for deployment.

## Dependencies
- Plan 07-01 complete (API client regenerated)
- Plan 07-02 complete (OIDC flow verified)
- Plan 07-03 complete (tests updated)
- Plan 07-04 complete (components implemented)
- Plan 07-05 complete (error handling implemented)
- All unit/integration tests passing

## Success Criteria
1. Complete OIDC login flow works:
   - User can log in with auth-service credentials
   - JWT token is obtained and decoded correctly
   - User is redirected to main page after login
2. Single-org user workflow verified:
   - No org dropdown visible in dialogs
   - No org dropdown visible in settings
   - Default org is used for all operations
   - User can create tasks without org selection
3. Multi-org user workflows verified:
   - Org dropdown visible in Add Task dialog
   - Org dropdown visible in Add Worktime dialog
   - Org dropdown visible in Company Settings (for admins)
   - Org context persists within dialogs
   - Org context cleared on dialog close
   - Org context persists on settings page
   - Creating task with org selection → task created in selected org
4. Member management workflows verified:
   - Admin can invite member to organization
   - Invited user receives invitation email (via auth-service)
   - Member can accept invitation and join org
   - Admin can view org members with correct status (ACTIVE or INVITED)
   - Admin can change member role (COMPANY_ADMIN ↔ USER)
   - Admin cannot remove self from organization
   - Admin cannot demote/remove last admin
   - Error messages show for permission/conflict errors
5. Error handling verification:
   - 403 errors show user-friendly message
   - 409 errors show appropriate message (already member, last admin)
   - 410 errors show expired link message (if applicable)
   - Network errors show connection message
6. All unit/integration tests pass: `npm run test:ci`
7. No compilation errors: `npm run build` succeeds
8. Browser console shows no auth-related errors or warnings
