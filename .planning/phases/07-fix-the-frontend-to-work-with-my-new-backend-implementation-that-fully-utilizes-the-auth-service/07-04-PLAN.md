1. Member management operations use new API shapes:
   - Member list calls use UUID userIds (not strings)
   - Invite dialog/form sends InviteMemberRequest via auth-service API
   - Role change dialog sends correct request to changeMemberRole endpoint
   - Member removal calls correct endpoint with UUID userId
   - All responses parsed as new MemberResponse shape
2. Multi-org UI implemented correctly:
   - Add Task dialog: org dropdown visible for multi-org users
   - Add Worktime dialog: org dropdown visible for multi-org users
   - Company Settings page: org dropdown visible for multi-org admins
   - All dialogs: org context cleared on close
   - Settings page: org context persists during page operations
3. No deprecated auth service methods used:
   - Audit confirms getUserOrganizationId() and getUserMemberships() not called
   - All org context comes from getActiveOrganization() or getOrganizations()
4. X-Org-ID header injection works:
   - Dialog operations send header from dialog org context
   - Settings operations send header from settings org context
   - Default: first org from JWT
5. Component tests pass with new API mocks (from Plan 07-03)
6. All components compile without TypeScript errors
