# Phase 07-03 Plan: Frontend Component Test Updates

## Goal
Update all frontend test fixtures (spec.ts files) to use the new API shapes identified in Plan 07-01 and prepare component tests for implementation changes in Plans 07-04 through 07-06. This plan focuses on test infrastructure and mocking patterns, not implementation changes.

## Dependencies
- Plan 07-01 complete (breaking changes documented)

## Success Criteria
1. All auth layer tests pass with new token format (authorities, user_id, orgs):
   - auth.service.spec.ts tests pass
   - auth.interceptor.spec.ts tests pass (token injection, X-Org-ID header logic)
2. All component tests use updated API mocks:
   - Task-related tests mock new TaskResponse shape (if changed)
   - Member-related tests mock MemberResponse with userId (UUID), accountId, status
   - WorkingTime tests mock AccountResponse correctly
3. Test fixtures for member management cover all scenarios:
   - Single member (ACTIVE)
   - Multiple members (mix of ACTIVE and INVITED)
   - Error scenarios (403, 409, 410 responses)
4. Multi-org scenarios tested in dialogs and settings:
   - Single-org user: org dropdown hidden
   - Multi-org user: org dropdown visible
   - Org context cleared on dialog close
   - Org context persists on settings page
5. All tests compile without TypeScript errors (no "cannot find name" etc.)
6. Test coverage improved or maintained (no regressions)
