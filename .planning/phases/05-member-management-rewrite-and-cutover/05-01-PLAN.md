<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| M2M token → management API | Backend service authenticates with client_credentials; end-user tokens must be rejected |
| End-user → management API | Direct user access to /api/v1/** must be blocked (no mgmt:admin scope) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-05-01-01 | Elevation of Privilege | DefaultSecurityConfig | mitigate | `.hasAuthority("SCOPE_mgmt:admin")` on all /api/v1/** except invitation status; end-user JWTs lack this scope |
| T-05-01-02 | Tampering | MembershipManagementController | mitigate | Last-admin guard in UserManagementService prevents removing/demoting last admin (returns 409) |
| T-05-01-03 | Information Disclosure | OrganizationManagementController | mitigate | GET /members requires M2M scope; only returns members of the specified org (no cross-org leakage) |
</threat_model>
