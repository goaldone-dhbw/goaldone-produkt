**Step 1 — Update VerificationTokenService.java:**

Add a second `@Value` field for password reset expiry and a helper method:
```java
@Value("${app.token.expiry-hours:24}")
private int defaultExpiryHours;

@Value("${app.token.password-reset-expiry-hours:1}")
private int passwordResetExpiryHours;

private int getExpiryHoursForType(TokenType type) {
    return type == TokenType.PASSWORD_RESET ? passwordResetExpiryHours : defaultExpiryHours;
}
```

| T-08-03-02 | Information Disclosure | GET /reset-password token consumption bug | mitigate | Fix: use `checkToken()` in GET handler (non-consuming) — prevents token being consumed before POST, which would make all reset attempts fail |
| T-08-03-03 | Information Disclosure | POST /forgot-password email enumeration | accept | Already mitigated: controller always shows the same message regardless of whether email exists (enumeration protection comment in PasswordResetController.forgotPassword()) |
| T-08-03-04 | Spoofing | Reset token interception | mitigate | 1-hour expiry (D-05) limits window; token is cryptographically secure (SecureRandom 32 bytes Base64) |
