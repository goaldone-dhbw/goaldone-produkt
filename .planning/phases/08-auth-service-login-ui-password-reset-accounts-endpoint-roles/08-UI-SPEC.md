# Phase 8 — UI Design Contract

> Visual and interaction contract for the auth-service Thymeleaf pages: login, forgot-password, reset-password, reset-success, and password-reset email templates.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope:** Auth-service native HTML only (Spring Boot Thymeleaf). Angular frontend is out of scope.
> **Not covered:** `/me/organizations` endpoint (pure API — no UI changes).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (GoalDone custom CSS — `base.css`) |
| Preset | not applicable |
| Component library | none — custom CSS variables in `static/css/base.css` |
| Icon library | Unicode/emoji for password toggle (👁 / 🙈); no external icon library |
| Font | System stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif` |

**Source:** `auth-service/src/main/resources/static/css/base.css` (verified by codebase scan)

**Framework:** Thymeleaf layout dialect. All pages (including `login.html`) MUST use:
```html
layout:decorate="~{fragments/layout}"
```
No page may have standalone `<body>` styles or inline gradient backgrounds. Inline `style=` attributes are permitted only for layout micro-adjustments not addressable by base.css classes (e.g., `style="width: 100%"` on a full-width button, `style="position: relative"` on a password field wrapper).

## Interaction Contracts

### Password Visibility Toggle

Required on: `login.html` (password field), `reset-password.html` (both password fields).

**HTML structure:**
```html
<div class="form-group" style="position: relative;">
    <label for="password" class="form-label">Password</label>
    <input type="password" id="password" name="password" class="form-input" required>
    <button type="button"
            onclick="togglePassword('password', this)"
            style="position: absolute; right: 10px; top: 32px; background: none; border: none; cursor: pointer; padding: 4px; color: var(--text-muted);"
            aria-label="Show password">
        👁
    </button>
</div>
```

**JavaScript (inline `<script>` at bottom of each page's content fragment):**
```javascript
function togglePassword(fieldId, btn) {
    var input = document.getElementById(fieldId);
    var isHidden = input.type === 'password';
    input.type = isHidden ? 'text' : 'password';
    btn.textContent = isHidden ? '🙈' : '👁';
    btn.setAttribute('aria-label', isHidden ? 'Hide password' : 'Show password');
}
```

**States:**
- Password hidden (default): button shows 👁, `aria-label="Show password"`
- Password visible: button shows 🙈, `aria-label="Hide password"`

**For `reset-password.html`:** Use two separate toggle buttons, each calling `togglePassword('password', this)` and `togglePassword('confirmPassword', this)` respectively.

---

### Form Validation UX

**HTML5 native validation only** — no custom JavaScript validation framework.

- `required` on all form inputs
- `minlength="12"` on password fields in reset-password.html
- `type="email"` on email input in forgot-password.html (browser validates format)
- Browser default validation popups are acceptable (no custom error UI required)

**Server-side errors** display via Thymeleaf conditional alerts (`.alert .alert-error`). No inline field-level validation indicators needed for Phase 8.
