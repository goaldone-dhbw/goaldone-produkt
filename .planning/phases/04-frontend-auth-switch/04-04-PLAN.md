  <name>Task 4: Update main page to display user identity and org memberships</name>
  <files>frontend/src/app/features/mainpage/mainpage.component.ts</files>
  <action>
Update the main/landing page to display user's authenticated identity, email, roles, and organization memberships. This confirms the user is logged in and shows what orgs they have access to.

Changes:

1. **Inject dependencies:**
   ```typescript
   private authService = inject(AuthService);
   ```

2. **Load user info in ngOnInit():**
   ```typescript
   ngOnInit() {
     const decodedToken = this.authService.getDecodedAccessToken();
     if (decodedToken) {
       this.userEmail = decodedToken['primary_email'] || decodedToken['email'];
       this.userId = decodedToken['user_id'] || decodedToken['sub'];
     }
     
     this.userOrgs = this.authService.getOrganizations();
     this.userRoles = this.authService.getUserRoles(); // { orgId: ['ROLE_ADMIN', ...] }
     this.activeOrg = this.authService.getActiveOrganization();
   }
   ```

3. **Template display (example):**
   ```html
   <div class="user-info" *ngIf="userEmail">
     <h2>Welcome, {{ userEmail }}</h2>
     <p>User ID: {{ userId }}</p>
     
     <div class="org-memberships">
       <h3>Your Organizations</h3>
       <ul>
         <li *ngFor="let org of userOrgs">
           <strong>{{ org.slug }}</strong>
           <span class="role">{{ org.role }}</span>
           <span class="id">({{ org.id }})</span>
         </li>
       </ul>
     </div>
     
     <div class="roles-by-org">
       <h3>Roles</h3>
       <div *ngFor="let orgId of objectKeys(userRoles)">
         <strong>{{ findOrgName(orgId) }}:</strong>
         <span *ngFor="let role of userRoles[orgId]">{{ role }}</span>
       </div>
     </div>
   </div>
   
   <div *ngIf="!userEmail" class="not-authenticated">
     <p>Not authenticated. Click login button.</p>
   </div>
   ```

4. **Helper methods:**
   ```typescript
   objectKeys(obj: any): string[] {
     return Object.keys(obj);
   }
   
   findOrgName(orgId: string): string {
     return this.userOrgs.find(org => org.id === orgId)?.slug || orgId;
   }
   ```

Per D-12: First Login Default — active org and org context loaded and displayed.
Per D-07: Roles in Member Lists Only (this is a confirmation view, not a member list, so role display is OK).
