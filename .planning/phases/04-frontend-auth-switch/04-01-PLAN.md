<interfaces>
From Phase 2 JWT Contract (02-02-SUMMARY.md):
- JWT `authorities` claim: flat string set of roles (e.g., ["ROLE_ADMIN", "ROLE_MEMBER"])
- JWT `user_id` claim: unique identifier (string)
- JWT `orgs` claim: array of org membership objects (e.g., [{ "id": "uuid", "slug": "string", "role": "string" }])
- JWT `primary_email` claim: user's primary email address
- Issuer: configurable via env.js (auth-service URL)
- Scopes: `openid profile email offline_access` (flat, no Zitadel URNs)

From AuthService (current implementation):
- initialize(): Promise<boolean> — loads discovery doc, restores token from storage
- getAccessToken(): string — returns current access token
- getDecodedAccessToken(): any — decodes JWT payload
- getUserRoles(): string[] — CURRENTLY extracts from Zitadel URNs (to be updated in Plan 02)
- getUserMemberships(): any[] — extracts from `orgs` claim
</interfaces>


4. **Keep the error handling for refresh tokens:** The existing pipe that handles `token_refresh_error` and `token_error` is still valid and should remain.
