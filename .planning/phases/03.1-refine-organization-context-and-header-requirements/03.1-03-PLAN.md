<task type="auto">
  <name>Task 1: Liquibase Schema Migration</name>
  <files>backend/src/main/resources/db/changelog/changes/012-finalize-user-org-schema.xml</files>
  <action>
    - Create a new Liquibase changelog file (012-finalize-user-org-schema.xml) that:
      1. Removes `zitadel_sub` and `zitadel_org_id` columns from the `user_accounts` (soon to be `memberships`) table.
      2. Adds `auth_user_id` column to the `user_identities` (soon to be `users`) table with a UNIQUE constraint.
      3. Migrates existing data: For each row in `user_accounts`, copy its `zitadel_sub` to the corresponding `user_identities.auth_user_id` (match via existing foreign key).
      4. Adds an INDEX on `auth_user_id` for login latency optimization.
      5. Adds a composite UNIQUE constraint on `(user_id, organization_id)` in `user_accounts` to enforce one membership per user per org.
    - The changelog must support both H2 (local) and PostgreSQL (dev/prod) dialects.
  </action>
  <verify>
    <automated>./mvnw clean compile && ./mvnw liquibase:update -Dspring-boot.run.profiles=local</automated>
  </verify>
  <done>Schema migration runs cleanly; user identity is indexed; memberships are unique per org.</done>
</task>
